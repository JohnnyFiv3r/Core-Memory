## Mem.beads — Project Planning Complete
- Full project plan at `plans/mem-beads-plan.md`
- Concept: persistent causal agent memory with lossless compaction
- 4-layer architecture: in-session → rolling window (10k token budget) → long-term (MEMORY.md) → archive (JSONL, forever)
- Key design decisions from planning session:
  - Every turn is a bead (unbroken causal chain)
  - Promotion is a status, not a type (any bead except session lifecycle can be promoted)
  - Compaction = conditional compression, never deletion ("information can't be destroyed, just converted")
  - Memory sub-agent as "librarian" — flags for promotion with written rationale
  - Rolling window dynamically generated at session start, fills up to 10k tokens (recency × promotion status)
  - session_start links to prior session_end (cross-session causal chain)
  - User authority trumps agent inference for promotion decisions
  - Future: myelination (frequent recalls strengthen beads), pruning (unused beads demote), association crawler, graph DB for long-term
- Inspired by Steve Yegge's beads concept, extended with human memory abstraction (sensory → working → long-term, with consolidation during "sleep"/session-end)
- 22 tickets created in beads tracker (epic: workspace-k1s) with full dependency chains
- Target: dogfood on this OpenClaw instance, then Line Lead (Spring AI), then Delta Bravo (PydanticAI)
- Python for scripts (lingua franca for AI tooling, aligns with Delta Bravo)
